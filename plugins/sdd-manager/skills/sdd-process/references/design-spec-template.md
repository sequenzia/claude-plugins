## Section Guidelines

### User Flows
- Map complete journeys, not just happy paths
- Include error and edge case flows
- Show decision points clearly
- Document success criteria for each flow

### Component Inventory
- Be exhaustive - list all UI components
- Include state variations
- Document props/configuration
- Show visual structure (ASCII wireframes)

### Wireframe Descriptions
- Write detailed enough for implementation
- Include responsive variations
- Specify content constraints
- Note scroll behavior

### Interaction Patterns
- Document all interactive states
- Specify timing and easing
- Include accessibility considerations
- Note platform variations if applicable

### Accessibility
- WCAG compliance is not optional
- Document keyboard navigation completely
- Include screen reader announcements
- Note reduced motion alternatives

---

## Detail Level Variations

### High-Level Design Spec
Include:
- Overview
- Design Principles
- User Flows (primary only)
- Component Inventory (major components)
- Open Questions

### Mid-Level Design Spec
Include all High-Level sections plus:
- Complete User Flows
- Full Component Inventory
- Wireframe Descriptions (key screens)
- Interaction Patterns (core patterns)
- Accessibility Considerations (overview)

### In-Depth Design Spec
Include all sections with full detail:
- Comprehensive user flows with error paths
- All components with state variations
- Complete wireframe descriptions
- Detailed interaction specifications
- Full accessibility documentation
- Visual design tokens

---

## Writing Guidelines

### For Developers
- Be specific about behavior
- Include edge cases
- Document state transitions
- Provide clear acceptance criteria

### Visual Documentation
- Use ASCII diagrams for wireframes
- Include measurements where critical
- Document responsive breakpoints
- Note animation timing

### Consistency
- Use consistent terminology
- Follow established patterns
- Reference design system components
- Link to related flows/components
